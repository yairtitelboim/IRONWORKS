# Memphis Colossus change layer (GEE)

The **Memphis Colossus Change (2023→2024)** map layer is generated by `memphis_geoai_change.py`. The script lives in MEM (copied from CALP) so you can re-run it with the **correct AOI** (5420 Tulane Rd = xAI Colossus site).

## Correct location for map overlay

- **Address:** 5420 Tulane Road, Memphis, TN (xAI Colossus)
- **Center (lon, lat):** `-90.0348674`, `34.9979829`

The existing GeoJSON was produced with a different AOI center, so the colored change polygons did not overlap the orange circle on the map. Re-run the script with the center above to fix that.

## Run from MEM repo root

**Requirements:** `earthengine-api` (and GEE auth), `alphaearth_api` at repo root (MEM has it).

```bash
cd /Users/yairtitelboim/Documents/Kernel/ALLAPPS/MEM

python scripts/memphis_geoai_change.py \
  --site-id memphis_colossus \
  --site-name "xAI Colossus - Memphis" \
  --center -90.0348674 34.9979829 \
  --radius-m 5000 \
  --out-dir public/data/memphis_change \
  --pairs 2023-2024 \
  --batch-tiles 2 \
  --overwrite
```

**GEE 5000-feature limit:** GEE aborts if a single request has more than 5000 features. Two ways to get full AOI coverage:

1. **Batching (recommended):** `--batch-tiles 2` splits the AOI into 2×2 tiles, exports each tile (≤5000 features), then merges. You keep `--scale 20` and get full detail with no truncation.
2. **Larger scale:** `--scale 30` or `--scale 40` yields fewer, larger polygons so one request stays under 5000 (simpler but coarser).

Output (same filename the map layer already loads):

- `public/data/memphis_change/memphis_colossus_2023-01-01_2023-12-31__2024-01-01_2024-12-31.geojson`
- `public/data/memphis_change/memphis_colossus_2023-01-01_2023-12-31__2024-01-01_2024-12-31_stats.json`

After this, the change polygons and the dashed orange circle (Colossus site) will align.

## Options

- **Batch tiles:** `--batch-tiles 2` (default 1). Use 2 for 2×2 tiles so each export stays under 5000 features; results are merged for full AOI. Keeps `--scale 20` and full detail.
- **Scale:** `--scale 20` (default). Use 30 or 40 only if you prefer a single request and coarser polygons instead of batching.
- **Max features:** `--max-features 5000` (default). Per-tile cap; do not increase (GEE limit).
- **Radius:** `--radius-m 4000` for a tighter AOI, or `6000` for wider.
- **Other windows:** e.g. `--pairs 2024-2024` (same year) or use `--window1 2024-01-01 2024-03-31 --window2 2024-10-01 2024-12-31` for custom ranges.
- **AOI from file:** `--aoi-geojson path/to/polygon.geojson` (Polygon or MultiPolygon) instead of `--center` / `--radius-m`.
